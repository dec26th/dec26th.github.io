  • ArrayList 与 LinkedList 的区别

    • 区别：ArrayList类比动态数组，LinkedList类比链表。

    • 插入元素时间复杂度

      对于ArrayList来说，主要消耗的时间在于 System.arraycopy动作。add(E) 时间复杂度为O(1),而对于add(E,index)而言，时间复杂度为O(n)。

      对于LinkedList来说，主要消耗的时间在于 for 循环找到指定的index。add(E) ** 时间复杂度为O(1),对于add(E,index)** 时间复杂度为O(n)。

    • ArrayList扩容的时机

      扩容的时机：新增元素后的minCapacity超过了当前集合的容量，则调用grow方法扩容，一般扩容到原大小的1.5倍。

    • ArrayList缩容的时机

      缩容的时机：ArrayList本身并不会自动缩容，但是提供了给外部缩容的trimToSize()方法。

    • ArrayList是否线程安全，有哪些线程安全的List

      否，Vector：给几乎所有的方法加上了synchronized关键字。所以被抛弃。

      ​ CopyOnWriteArrayList:加了写锁的ArrayList，但是读操作可以并发。

3. 影响效率的因素（负载因子，hash数组的size）

   **初始容量** 与 **负载因子**。**初始容量** 只是哈希表在创建时的容量。**负载因子**时哈希表在其容量自动增加之前可以到达多满的一种尺度。 **负载因子**过高会减少空间成本(减少扩容机制的触发概率), 但是会增加查询成本。 **初始容量** 也需要考虑条目数及其加载因子，最大程度上减下扩容机制的触发。

4. 扰动函数的作用

   扰动函数即`hash()`，其目的减少碰撞问题。

   原理：通过将高半区与低半区的异或，加大低位的随机性，同时也能一定程度上保留高位的信息。

   



5. 底层

   在jdk1.8以后，**HashMap**采用**位桶 + 链表 + 红黑树**的数据结构。 **位桶**产生冲突，则将元素加入链表，若链表的长度超过了**阈值**，则将链表转换为了**红黑树**(大大缩减了查找时间，O(lgn))。

  • HashMap 与 ConcurrentHashMap

    1. 为什么不安全？举例子

      1. get方法可能因为 resize（）触发了闭环结构。

        当两个线程A,B同时调用了HashMap的resize()方法时，则可能会在A调用方法还没完成后，时间片结束，B调用resize()方法并且将HashMap的容量扩大，但是当A获得CPU控制权之后，A所持有的HashMap是已经扩容过后的HashMap，而如图所示，A首先将[3,A]迁移到新数组，然后再处理[7,B]，而处理完[7,B]后需要处理[7,B]的next，而在这时，经过线程B之后，[7,B]的next已经变为了[3,A]，所以这将生成一个[7,B]与[3,A]的环形链表。

      2. 在有多个put线程时，会产生数据被覆盖的情况。假设A先通过hash获得了桶的索引值(无碰撞)，但是时间片结束，此时B刚好获得相同的桶索引值(A还没插入，依旧是无碰撞)并且插入，但到了A运行后，他并不知道桶已经有了B会发生碰撞，依旧写入桶中的相应索引位置，所以B的数据被覆盖。

  • HashMap 与 HashTable

    1. 数据结构

      HashTable : 数组 + 链表

      HashMap : 数组 + 链表 + 红黑树

    2. 父类

      HashTable 的父类是Dictionary类

      而 HashMap 的父类是 Map类

    3. 扩容

      HashTable 初始容量为1，增大方式为 old * 2 + 1

      而 HashMap 初始容量为16，容量一定为2的倍数

      HashTable 计算 hash值 采用的取模

      而 HashMap采用的是按位与(本质也是取模）

    4. 线程上锁的范围T

      HashTable 的 put()方法是synchronized的。

  • euqals 与 ==

    对于引用变量而言，使用==判断的是两个引用变量是否引用的是同一个对象，而equals默认也是==判断，但是可以根据用户的需求重写该方法。

    • 为什么要重写hashCode（）

      需要对比的时候，首先用hashCode去对比，如果hashCode不一样，则表示这两个对象肯定不相等（也就是不必再用equals()去再对比了）,如果hashCode相同，此时再用equals()比，如果equals()也相同，则表示这两个对象是真的相同了，这样既能大大提高了效率也保证了对比的绝对正确性！

  • String StringBuilder 与 StringBuffer

    • 区别与使用场景

      • String

        字符串常量，字符串长度不可。

      • StringBuilder

        字符串变量，线程不安全，性能相较于StringBuffer好一些。

      • StringBuffer

        字符串变量，并且线程安全，性能相较于StringBuilder差一些。

        append()方法为synchronized

    • 为什么String设置为不可变对象

      • 字符串常量池

        字符串常量池(String pool, String intern pool) 是Java 堆内存中的特殊存储区域，当创建一个新的String对象时，假如内容已经存放在字符串常量池中，则不需要new，直接引用该已经存在的对象即可。

      • 安全性

        String被许多的Java类(库)用来当作参数，URL,文件路径path，反射机制的String参数等等，若String可变，则可能引发不可知的问题。

      • 默认的多线程安全

      • 创建的时候hashcode就已经确定，不需要重新计算，这也HashMap经常将String当作key的原因(hashcode不需要重复计算)。

  • 重写与重载

    • 重写

      重写是子类对父类可以访问方法(方法名，参数列表都必须相同，返回值可以小于等于父类返回值)进行重新的声明。属于晚期绑定。

    • 重载

      重载是对指具有相同方法名的方法带有不同参数列表(参数个数不同或者至少一个参数类型不相同)的版本，其返回值也可以不同。

      仅返回类型不同，其余参数都相同的方法不是合法的重载方法。

  • 深拷贝与浅拷贝
    • 深拷贝：将所有的属性都拷贝进一个新的对象当中(该对象与原对象的地址不同，不是同一个对象)。
    • 浅拷贝：返回当前对象的引用(返回的是对象本身的引用)。
  • Java三大特性
    • 封装：将成员变量与方法封装进一个类体当中，仅对外公开可访问的方法与成员变量。
    • 继承：Java支持单继承。支持在现有类的基础上，增加新的方法或重写已有的方法，从而产生一个新的类。
    • 多态：通过引用变量调用实例函数式，根据所引用的实际对象的类型，执行该类型的相应实例方法，从而表现出不同的行为。一般通过继承时覆盖父类的实例方法实现多态，在运行时根据引用变量指向的实际对象类型，来重新计算调用方法的入口地址。
